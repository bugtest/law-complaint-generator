## 1. 项目概述

### 1.1 项目背景

律师在起诉过程中需要将原始证据材料（PDF 格式）和整理后的文档（Word 格式）转换为符合国家标准的要素式起诉状。本系统旨在通过 AI 技术自动化这一过程，提高律师工作效率。

### 1.2 目标用户

- 执业律师
- 律所助理人员
- 多用户协作环境

### 1.3 核心价值

- 自动化提取案件要素，减少手动录入
- 模板化管理起诉状格式，确保符合国家标准
- 本地化部署，保障律师数据安全

## 3. 功能模块设计

### 3.1 文档上传与解析模块

**功能描述**: 支持律师上传 PDF 证据文件和 Word 整理文档，并自动解析提取文本内容。

**处理流程**:

1. **PDF 处理**
   - 文字版 PDF → 使用 pdfplumber 直接提取文本
   - 扫描件 PDF → 使用 PaddleOCR 进行 OCR 识别
   - 提取原始文本和基础结构（段落、表格）

2. **Word 处理**
   - 使用 python-docx 提取整理后的文本内容
   - 保留格式信息（标题、段落样式）

3. **双文档对比展示**
   - 左侧显示原始证据（PDF 解析结果）
   - 右侧显示整理内容（Word 解析结果）

### 3.2 AI 要素提取引擎

**功能描述**: 利用 AI 技术从解析后的文档中自动提取起诉状所需的结构化要素。

**输入**: PDF 原始文本 + Word 整理文本

**提取要素类别**:

| 要素类别 | 字段说明 |
|----------|----------|
| 当事人信息 | 原告/被告姓名、身份证号、地址、联系方式 |
| 诉讼请求 | 逐项提取具体请求内容 |
| 事实与理由 | 归纳时间线、关键事实、法律依据 |
| 证据清单 | 识别证据名称、证明目的、页码 |

**输出**: 结构化的 JSON 数据，可供律师审核编辑

**特性**:
- 支持人工校正
- 校正结果可用于后续模型优化
- 提供 AI 置信度评分

### 3.3 模板管理模块

**功能描述**: 允许用户上传、管理起诉状 Word 模板，支持版本控制和占位符映射。

**核心功能**:

1. **模板上传与存储**
   - 支持多个模板同时存在
   - 可设置默认模板

2. **模板版本管理**
   - 每次上传保留历史版本
   - 支持版本回溯

3. **占位符解析**
   - 识别 `{{字段名}}` 语法
   - 建立模板字段与系统要素的映射关系

4. **模板选择**
   - 生成文书时可选择使用哪个模板

### 3.4 文书生成模块

**功能描述**: 将 AI 提取的要素填充到选定的 Word 模板中，生成符合国标的要素式起诉状。

**处理流程**:

1. 读取用户确认的要素数据
2. 加载选定的 Word 模板
3. 将要素填充到模板占位符
4. 生成最终 Word 文档
5. 支持在线预览和编辑
6. 支持导出为 Word 或 PDF 格式

### 3.5 用户与权限模块

**功能描述**: 多用户账户管理和数据隔离。

**核心功能**:

- 用户注册与登录（JWT 认证）
- 多用户账户管理
- 数据隔离：每个律师只能访问自己的案件和文书
- 基础操作日志记录

## 7. 页面流程设计

### 7.1 页面结构

```
登录页
  │
  ▼
案件列表页 ────→ 创建案件 ────→ 上传文档 (PDF+Word)
    │                                  │
    │                                  ▼
    │                          解析中 (进度显示)
    │                                  │
    │                                  ▼
    │                          要素审核页 (可编辑)
    │                                  │
    │                                  ▼
    │                          选择模板 → 生成预览
    │                                  │
    │                                  ▼
    └──────────────────────────  导出下载 (Word/PDF)
```

### 7.2 关键页面

1. **登录页**
   - 用户名/密码登录
   - 记住登录状态

2. **案件列表页**
   - 显示所有案件（名称、状态、创建时间）
   - 支持搜索、筛选
   - 支持新建案件、删除案件

3. **案件详情页**
   - 显示案件基本信息
   - 显示已上传文档列表
   - 显示已生成文书列表
   - 操作入口：上传文档、解析、生成文书

4. **文档上传页**
   - 支持拖拽上传
   - 区分 PDF（证据）和 Word（整理文档）
   - 显示上传进度

5. **要素审核页**
   - 左右分栏布局
   - 左侧：原始文档预览
   - 右侧：结构化表单
     - 原告信息表单
     - 被告信息表单
     - 诉讼请求列表（可增删改）
     - 事实与理由文本域
     - 证据清单列表（可增删改）
   - 提交按钮：确认要素，进入生成环节

6. **模板管理页**
   - 模板列表（名称、版本、上传时间）
   - 上传新模板
   - 删除模板
   - 设置默认模板

7. **文书预览页**
   - 在线预览生成的 Word 文档
   - 支持二次编辑
   - 导出为 Word 或 PDF

## 8. 错误处理设计

### 8.1 错误场景与处理策略

| 场景 | 错误信息 | 处理策略 |
|------|----------|----------|
| PDF 无法解析（加密/损坏） | "文件无法解析，请检查文件是否损坏或加密" | 返回明确错误，提示重新上传 |
| OCR 识别失败 | "OCR 识别失败，请检查图片质量" | 降级为手动录入模式 |
| OCR 质量低 | "识别质量较低，请确认内容准确性" | 警告提示，建议人工核对 |
| AI 提取超时 | "要素提取中，请稍候查看结果" | 异步处理 + 轮询机制 |
| 模板占位符不匹配 | "模板中有 N 个字段未找到对应数据" | 警告提示，显示未匹配字段列表 |
| 文书生成失败 | "文书生成失败，请重试" | 保留已填充数据，支持重新生成 |
| 文件过大 | "文件大小超过限制（XX MB）" | 拒绝上传，提示压缩文件 |
| 不支持的文件格式 | "仅支持 PDF/DOC/DOCX 格式" | 拒绝上传，提示正确格式 |

### 8.2 错误响应格式

```json
{
    "error": {
        "code": "PARSE_FAILED",
        "message": "文件无法解析，请检查文件是否损坏或加密",
        "details": {
            "file_type": "pdf",
            "file_name": "evidence.pdf"
        },
        "suggestion": "请重新上传文件或联系技术支持"
    }
}
```

## 9. 安全设计

### 9.1 数据安全

- 文件存储加密（可选配置）
- 数据库敏感字段加密（如联系方式、身份证号）
- 本地部署，数据不出用户环境
- 定期备份机制

### 9.2 访问控制

- JWT Token 认证，24 小时过期
- 用户数据隔离（通过 user_id 过滤）
- 密码 bcrypt 加密存储
- 密码强度校验（最少 8 位，包含字母和数字）

### 9.3 文件安全

- 上传文件类型校验（仅允许 PDF/DOC/DOCX）
- 文件大小限制（默认 50MB，可配置）
- 文件存储路径与用户隔离
- 文件上传病毒扫描（可选）

### 9.4 审计日志

- 记录登录、上传、生成、下载等操作
- 日志包含：用户 ID、操作类型、资源类型、时间戳、IP 地址
- 日志仅管理员可见

## 11. 部署与运维

### 11.1 部署要求

- Python 3.10+
- Node.js 18+ (前端构建)
- SQLite 或 PostgreSQL
- 推荐配置：4 核 CPU / 8GB 内存 / 100GB 存储

### 11.2 配置文件

系统通过 `.env` 文件管理配置项：

```bash
# 数据库
DATABASE_URL=sqlite:///./law_system.db
# 或 DATABASE_URL=postgresql://user:pass@localhost/law_system

# JWT 配置
JWT_SECRET_KEY=your-secret-key
JWT_EXPIRE_HOURS=24

# 文件存储
FILE_STORAGE_PATH=/var/law_system/files
MAX_FILE_SIZE_MB=50

# AI 配置
ANTHROPIC_API_KEY=your-api-key
AI_MODEL=claude-sonnet-4-20250514

# OCR 配置
OCR_ENABLED=true
OCR_LANGUAGE=ch

# 日志
LOG_LEVEL=INFO
LOG_FILE=/var/law_system/logs/app.log
```

### 11.3 启动脚本

```bash
# 后端启动
cd backend
source venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8000

# 前端启动
cd frontend
npm install
npm run build
# 或使用开发服务器
npm run dev
```
